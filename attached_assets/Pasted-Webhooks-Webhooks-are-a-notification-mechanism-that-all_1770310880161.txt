Webhooks
Webhooks are a notification mechanism that allows platforms to listen for events that can occur asynchronously.

Before starting, you need to setup an endpoint at your server that accepts: post, put or patch. If you need something for testing, you can use https://webhook.site/.

All webhooks are managed through the webhook endpoint:

/v1/webhooks

The receiver is determined based on the key that is used, either platform or entity.

Registering Webhooks
To register a webhook, please checkout the attached Postman collection.

Timeouts
The webhook client will timeout whenever one of the following durations are exceeded:

Connect: 10 seconds
Write: 5 seconds
Read: 10 seconds
If your server takes more that 10 seconds to finish its job, please put the job on a queue and respond to the webhook client as soon as possible.

Webhook Lifecycle
When a webhook succeeds, an attempt will be logged and the Webhook Attempts log and it is done.

If a webhook fails, then Moyasar will retry the webhook event four more times, making it five attempts including the first one.

Here is the timing for the retries:

Attempt Number	Delay Time
1	60 seconds
2	600 seconds = 10 minutes
3	1800 seconds = 30 minutes
4	3600 seconds = 1 hour
Webhook Attempts Log
Moyasar servers will keep track of all webhook attempts for an arbitrary amount of time (they will be cleaned without any notice, they are used for development purposes and not for production).

To view webhook attempts (in descending order), make a get request to the following using the receiver key (platform or entity):

/v1/webhooks/attempts

Each attempts will have a result data field that indicates whether it succeeded or failed:

success: the webhook was delivered successfully
dns_error: cannot resolve hostname because of a DNS error
connection_error: cannot establish a connection to the server
connection_timeout: Connection timed out
malformed_response: invalid response was returned
tls_error: cannot establish a TLS connection
response_timeout: request was sent but read (response) timed out
response_error: the server did not return a 2xx response
invalid_url: invalid webhook URL
